# 金字塔原理 Skill

## 概述

本技能将金字塔原理的方法论转化为可执行的思维和工作流程。核心方法论详见 [references/concepts.md](references/concepts.md)，常见场景示例见 [references/examples.md](references/examples.md)。

## 契约

- **输入**: 用户的文本内容、问题描述、需要重构或诊断的素材
- **输出**: Markdown 格式的金字塔结构文档 / 结构化诊断评分报告
- **边界**:
  - 本技能审查逻辑结构，**不核查**内容的事实正确性
  - 非中文内容的 MECE 分析与 SCQA 检测效果有限
  - 不处理图形化流程图/脑图的可视化排版
- **脚本失败回退**: 若 `analyze.py` 或 `restructure.py` 执行失败，回退到人工分析模式

## 安全规则

⚠️ 使用脚本前必须获得用户确认。**绝对禁止**在未告知用户的情况下:
- 覆盖已有文件（应使用唯一命名）
- 删除或修改用户的原始内容
- 将用户内容写入非临时位置

## 六大工作流

### 1. 诊断现有内容

**触发**：用户粘贴一段文字让你"分析是否符合金字塔原理"或"诊断结构"。

1. 将内容写入临时文件
2. 运行 `python scripts/analyze.py <file>` 获得评分报告
3. 解读报告并给出具体的结构化改进建议

### 2. 重构杂乱内容

**触发**：用户说"帮我按金字塔原理重新组织这段内容"。

1. 将用户的内容写入临时文件
2. 运行 `python scripts/restructure.py <file>` 获得初步重构
3. 人工打磨：补充 MECE 归类、调整层级深度、添加逻辑递进
4. 交付用户可用的金字塔结构文档

### 3. 从零构建金字塔

**触发**：用户说"我想用金字塔原理写一份关于[主题]的报告/提案"或"帮我用金字塔原理梳理[问题]"。

**流程（自上而下法，推荐）：**

1. **确定中心思想**：与用户确认核心结论（一句话）
2. **搭建第一层支撑**：MECE 分解出 3-7 个主要论点
3. **逐层下沉**：对每个主要论点，继续问"为什么"或"具体是什么"
4. **加工序言**：用 SCQA 框架写开场白
5. **交付**：Markdown 格式的金字塔结构

检查清单：
- [ ] 结论是否先行且唯一？
- [ ] 每一层是否是对下层的概括？
- [ ] 同一层子项是否 MECE？（无重叠、无遗漏）
- [ ] 同级论点是否按逻辑顺序排列？（时间/结构/程度）
- [ ] 层级深度是否适中？（通常 3-4 层）

### 4. 应用 MECE 分析框架

**触发**：用户说"用 MECE 帮我分析[问题]"或"分析这个问题的根因"。

1. 确定分析对象（问题/决策/方案）
2. 选择适用的 MECE 框架（见 `references/concepts.md` 常用框架表）
3. 逐层分解，确保每层不重叠不遗漏
4. 检查是否有"其他"类——尽量拆解掉

### 5. 撰写 SCQA 序言

**触发**：用户说"帮设计开场白"或"用 SCQA 框架写序言"。

1. 确认 4 要素：
   - **S (Situation)**：读者已知且认同的背景事实
   - **C (Complication)**：发生了什么变化/问题/冲突
   - **Q (Question)**：因此产生的关键问题
   - **A (Answer)**：你的核心观点/解决方案
2. 根据场景选择变体（标准式/开门见山式/突出忧虑式/对比式）
3. 交付序言段落

### 6. 审查与优化

**触发**：用户说"帮检查逻辑"或"审查这个论证"或"我的论证有漏洞吗"。

1. 识别内容中的论点层级
2. 逐层审查：
   - 上层是否准确概括了下层？
   - 下层是否充分支撑了上层？
   - 横向论点之间是否 MECE？
   - 推理方式（演绎/归纳）是否合理？
3. 指出逻辑漏洞并给出修正方案

## 脚本使用

```
# 诊断分析
python scripts/analyze.py input.txt

# 重构内容
python scripts/restructure.py input.txt -o output.md

# 交互式构建金字塔
python scripts/restructure.py --interactive
```

## 实用建议

- **优先使用自上而下法**：先有结论再找论据，比自下而上法更高效
- **控制每层论点数量**：3-7 个为佳，少于 3 个考虑合并，多于 7 个考虑分组
- **横向关系统一**：同一层要么都用归纳推理，要么都用演绎推理，不要混用
- **序言必用 SCQA**：即使不显式标注，也要包含 S-C-Q-A 的四要素
- **检查"所以呢"**：每说完一层，问自己"所以呢？"——答案就是上一层
